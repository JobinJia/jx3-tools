# JX3-Tools UI 重设计 · 设计文档

日期：2026-06-10
状态：已与用户逐节确认通过（风格、导航、设计系统、三个页面线框）

## 1. 目标与范围

把当前「水墨元素 + naive-ui 默认风格」的混搭 UI 统一为**新中式**视觉语言：现代布局骨架 + 高浓度国风质感（宣纸、墨线、朱砂、宋体标题）。

**范围内**：主布局（侧栏导航）、改键页、MAC地址页、按键页的模板与样式；设计 token 体系；浅/深双主题。
**范围外**：一切业务逻辑（composables、stores、services、Rust 端）不动；树节点的选中/收藏/打开文件夹等交互行为不变；不新增路由。

唯一的小型功能新增：改键页「最近操作」记录（仅 localStorage，纯前端，复制成功时追加）。

## 2. 设计 token（CSS 变量 + naive-ui themeOverrides 共用一份）

所有颜色经由 CSS 变量定义在 `:root` / `.dark`，并映射进 naive-ui 的 `themeOverrides`，两套主题同构。

### 主题切换

跟随系统（`useOsTheme`）+ 侧栏底部手动切换按钮（三态：跟随系统/浅/深，存 localStorage）。naive-ui 用 `darkTheme` + 对应 themeOverrides；自定义样式靠根元素 `.dark` class 切 CSS 变量。

## 3. 通用视觉规则

- **字体**：页面标题宋体（`'Songti SC','STSong',serif`），字距 2~3px，标题尾随 8px 朱砂小方印；正文/控件系统字体（PingFang SC）；MAC 地址、热键等代码值用等宽（`ui-monospace,Menlo,monospace`）。
- **墨线**：页头分隔线统一为「浓→淡」横向渐变（`linear-gradient(90deg, var(--ink), transparent)`）。
- **纸纹理**：浅色主题页面底叠加 CSS 噪点纹理（SVG feTurbulence data-uri 或细噪点 PNG，透明度极低）；**移除现有古塔背景图**（`pagoda-8018757_1280.jpg`）与 MRicePaper 包裹（纹理改由内容区自己负责，shuimo-ui 仅保留全局注册不强依赖其视觉组件）。
- **卡片**：8px 圆角（大卡 10px）、1px `--line` 边框、极轻阴影 `0 1px 2px rgba(44,42,38,.04)`。
- **页头模板**（三页统一）：宋体标题 + 朱砂印 → 一行说明文字 → 渐变墨线；页头右侧为该页的工具/状态位。

## 4. 主布局（MainLayout）

- 侧栏 58px、底色 `--ink`（深色主题 `#1a1816`）。
- 导航项为 36×36 **印章字**按钮：改 / 网 / 按（宋体单字）。选中态：朱砂底 + 米色字 + 内描边；未选中：透明底 + 细描边 + 灰字。悬停浮出 tooltip 显示全名（改键 / MAC地址 / 按键）。
- 侧栏底部：主题切换圆钮（☀/☾/跟随）+ 版本号小字。
- 内容区直接渲染 router-view，页面自己负责页头；移除 MRicePaper 包裹与 n-layout 嵌套，主布局改为纯 div + flex（侧栏固定宽 + 内容区 flex-1）。
- 路由 meta 中的 emoji icon 字段改为 `sealChar: '改' | '网' | '按'`。

## 5. 改键页

布局：页头 + 三栏（源 270px / 箭头 24px / 目标 270px / 操作面板自适应 ≥170px）。

- **页头右侧**：路径胶囊 —— 📁 + 中段省略的路径 + 「切换」「⟳ 刷新」收纳其中（替代现有裸文本 + 两个 tiny 按钮）。
- **源卡片**：标题头「源角色」+ 右侧小分段控件（全部 / 常用 N）切换 SourceTree 与 TemplateList；下方搜索框、树。
- **目标卡片**：标题头「目标角色」+ 小字警示「键位将被覆盖」；下方搜索框、树。
- **两树之间**：垂直居中的朱砂箭头 ➜，表达复制流向。
- **操作面板**（替代现有 Result 组件的视觉）：
  - 「本次操作」卡：源名（朱砂标）→「↓ 覆盖」→ 目标名（黛蓝标）；朱砂大按钮「复 制 键 位」（宋体、字距 4px）；按钮下小字「目标角色现有键位将被覆盖，不可撤销」。未选齐源/目标时按钮禁用。
  - 「最近操作」卡：最多保留 10 条 `源 → 目标 + 时间`，localStorage 持久化，复制成功时追加。
- **空状态**（无 basePath）：居中卡片 —— 印章风图形 + 说明文字 + 朱砂主按钮「选择 userdata 目录」。
- 树节点行为（选中、⭐ 收藏、📂 打开文件夹、hover 显示图标）不变，仅按 token 调整配色。

## 6. MAC地址页

居中单列（max-width 430px）：

- **地址主卡**：居中展示 —— 小字「当前地址」→ 26px 等宽当前 MAC → 状态标签（原始=竹青 / 已修改=赭石）→ 小字原始地址 → 按钮行（朱砂「随机修改」+ 描边「还原地址」，loading 状态沿用）。
- **设置卡**：「重启自动还原」标题 + 说明小字 + 右侧开关（逻辑不变）。
- **帮助**：默认一行弱化文字 + 「展开说明 ▾」，展开后显示现有两段帮助内容。
- 错误仍用 alert，置于墨线下方。
- 删除现状的 n-table 信息表格与组件内重复定义的工具类 CSS（.flex/.gap-2 等，直接用 UnoCSS）。

## 7. 按键页

居中单列（max-width 480px）：

- **页头右侧状态徽章**：呼吸灯圆点 + 状态文字（运行中=竹青 / 已停止=灰 / 未注册=赭石）+ 分隔后小字「F11 开始 · F12 停止」（取当前配置值）。替代现有按钮排里的 n-tag。
- **配置卡**分两节：
  - 「按键行为」：按键模式（分段控件 全局/窗口，窗口旁 ⓘ 浮层放现有三条窗口模式说明；非 Windows 禁用并提示）、触发按键（键帽输入框）、触发频率（数字输入 + 「毫秒/次」后缀）。
  - 「控制热键」：节标题带小字「（软件后台时也生效）」；开始热键 / 结束热键（键帽输入框）。
  - 窗口模式时在两节间插入「目标窗口」行（过滤输入 + 刷新 + 下拉选择，逻辑与 5s 自动刷新不变）。
  - 底部操作条：朱砂「保存配置」+ 描边「停止任务」（禁用条件不变）。
- **键帽输入框**：等宽字体、`--paper-card-soft` 底、1px 边框且下边框 2px（物理键帽感）、聚焦时朱砂描边 + 显示「请按下按键…」；录入逻辑（keydown 捕获）不变。
- **帮助**：底部一行弱化文字「点击热键输入框后按下键盘即可录入 · macOS 需授权辅助功能」+「更多说明 ▾」展开现有完整说明；替代现有三段堆叠 alert。
- lastError alert 保留，置于墨线下方。

## 8. 实现要点

- 新增 `src/assets/theme.css`（或并入 main.css）：两套 CSS 变量、纸纹理、墨线/印章/键帽等通用 class。
- 新增 `src/composables/useTheme.ts`：三态主题（system/light/dark），输出 naive-ui theme + 根 class，localStorage 持久化。
- `App.vue` / 入口：`n-config-provider` 挂 theme + themeOverrides（现状若无 config-provider 则补上）。
- 字体仅用系统已有字体栈，不引入 webfont（包体积与离线考虑）。
- 页头可抽 `PageHeader.vue` 通用组件（title + description + 右侧 slot）。
- 现有 `useOsTheme` 的侧栏背景逻辑、古塔图片资源、MacId 组件内工具类 CSS 一并清理。
- shuimo-ui 组件（改键页收藏弹窗的 m-dialog/m-input/m-button 等）**保留不替换**，仅通过 CSS 变量微调配色使其与 token 协调；`createMUI` 注册保持现状。
- 验证：`pnpm type-check`、`pnpm lint`、`pnpm test:unit` 通过；`pnpm tauri:dev` 目检两套主题下三个页面。
